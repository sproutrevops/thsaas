TH SAAS — Platform Overview

TH SAAS is an internal data platform that connects Google Sheets and HubSpot CRM to give your team a unified view of MRR, headcount, and client movement across products and partners.

The live site runs in Static Mode — all data is fetched directly from Google Sheets via the Sheets API using your Google login. No server is required.

Static Mode — you sign in with Google once; your OAuth token is used to read the spreadsheet in real time. Settings (Sheets ID, access rules) are stored in your browser's localStorage.

Pages Reference

▣ Dashboard

The home screen. Shows three summary cards at a glance:

  • Sheet Tabs — how many tabs are found in the configured spreadsheet.
  • HubSpot Companies — total companies pulled from HubSpot (not available in static mode).
  • Server Status — shows Static Mode (always healthy in this version).

The Connection Status section shows whether Google Sheets and HubSpot are reachable. Quick action buttons let you jump to any page.

📦 Per Product Movement (PPM)

The main data table. Displays every client–product combination with monthly HC (headcount) and computed MRR side by side.

Setup — three sheets are required:

  • PPM Sheet — master client list with Partner, Client Name, Product, Application, Start Date, Renewal Date, and a base MRR column.
  • HC Sheet — headcount data per client/app per period. Columns follow the pattern HC YYYY-MM or MMM-YY. A headcount > 0 means the client is active that month.
  • Unit Price Sheet — per-seat pricing per partner/product. Used to compute MRR = HC × unit price for each month.

Column auto-detection: the page tries to match columns by common names (e.g. "Partner", "Client Name", "Application"). Use the dropdowns in the config row to override if your headers differ.

Filters:

  • Partner — filter the table to one partner.
  • Client — filter to one client within the selected partner.
  • Application search — free-text search across the Application column.

Table layout: the first four columns (Partner, Client Name, Start Date, Renewal Date) are sticky — they stay visible as you scroll right through the monthly data. Each month shows two sub-columns: HC and MRR. Rows are grouped by partner with a collapsible group header.

Stats bar (above the table) shows the totals for the current filter: Total MRR, Total HC, Active Clients, and Partners.

📈 SaaS Metrics

An MRR waterfall table — the standard SaaS cohort analysis. Each column is a month; rows show how MRR moved:

  • Beginning MRR — ending MRR from the prior month.
  • New Business — MRR from clients that were inactive last month and became active this month.
  • Expansion — MRR increase from clients that were already active.
  • Contraction — MRR decrease from clients that stayed active but reduced HC.
  • Lost / Churned — MRR from clients that went from active to zero HC.
  • Ending MRR — Beginning + New + Expansion − Contraction − Lost.

Source of truth: HC is the sole signal for client activity. A client is active when HC > 0. MRR for each period is computed as HC × unit price.

Tip: load Per Product Movement first — SaaS Metrics reuses PPM's loaded data. Use the Download CSV button to export the waterfall.

🏢 Company Movement

A high-level MRR movement summary rolled up across all products and partners — the same waterfall logic as SaaS Metrics but presented as a compact overview chart. Useful for board-level or QBR reporting. Also reuses PPM data (load PPM first).

📊 Google Sheets

A raw data browser. Lists all tabs in your spreadsheet; clicking a tab loads its contents as a paginated table. Useful for quickly verifying data without opening the spreadsheet.

🔶 HubSpot CRM

Displays company records pulled from HubSpot. Not available in static mode — requires the Node.js server version running locally with a HubSpot Private App token configured in .env.

⚙️ Settings

  • Spreadsheet URL or ID — paste the full Google Sheets URL or just the ID (the string between /d/ and /edit). Saved in localStorage.
  • HubSpot Access Token — Private App token (not used in static mode).
  • Authorized Emails — comma-separated list of emails allowed to sign in. Leave blank to allow any Google account.
  • Authorized Domain — allow everyone from a domain (e.g. sprout.ph). Leave blank to allow all domains.
  • Connection Status — test whether Sheets and HubSpot are reachable with the current credentials.

Google Sheets Setup Guide

Three sheets are required inside a single spreadsheet. Suggested tab names:

Sheet / Tab Purpose Key columns
PPM / Master Client–product list Partner, Client Name, Product, Application, Start Date, Renewal Date, MRR
HC / Headcount Monthly headcount per client Client Name, Application, HC YYYY-MM …
Unit Price Per-seat pricing per partner/product Partner, Client Name, Application, Unit Price
Important: your Google account must have at least Viewer access to the spreadsheet. In static mode the sign-in OAuth token is used — no service account is needed.

Data Logic & Calculations

Active vs Inactive clients

A client is considered active in a given month when their HC value is greater than zero in the HC sheet for that period. HC = 0 (or blank) means inactive / churned for that month.

Computed MRR

Monthly MRR per client is not taken directly from the PPM MRR column — it is computed as:

MRR (month) = HC (month) × Unit Price (from Unit Price sheet)

The base MRR column in the PPM sheet is used as a reference/fallback only.

SaaS Metrics categories

  • New Business — HC was 0 last month, HC > 0 this month → entire MRR is New.
  • Lost / Churn — HC was > 0 last month, HC = 0 this month → entire prior-month MRR is Lost.
  • Expansion — client remained active (HC > 0 both months) and MRR increased.
  • Contraction — client remained active and MRR decreased.

Troubleshooting

Symptom Likely cause & fix
Dashboard shows Not configured Go to Settings and verify the Spreadsheet ID is saved. The default ID is pre-filled but can be overwritten by a prior blank save.
PPM loads no data / columns are missing Select the correct sheet from the PPM Sheet dropdown, then click Load. Use the column dropdowns to map headers manually if auto-detect fails.
SaaS Metrics shows nothing PPM must be loaded first. Navigate to Per Product Movement, select sheets, click Load, then return to SaaS Metrics and click Refresh.
SaaS Metrics totals don't match Company Movement Both use the same data engine. If they differ, reload PPM to refresh the cache, then recalculate both pages.
Sign-in button does nothing Your browser may be blocking the Google popup. Allow popups for this site in browser settings and try again.
Stale data after spreadsheet changes Click the ↻ Refresh or ↻ Reload Tabs button on the relevant page. Each button re-fetches directly from the Sheets API.
